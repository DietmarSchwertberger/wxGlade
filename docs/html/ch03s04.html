Properties Window

The properties window let you see and edit the properties that applies to the element that you select. This window consist in three tabs. The first tab contains the common properties that apply to all widgets.

Figure 3.6. Common Properties

As shown in Figure 3.6 The common properties are related to name, size, colors and fonts.

The second tab is related to layout properties that controls position and resizing inside the sizer.

Figure 3.7. Layout Properties

These properties apply to any widget. You can check or uncheck any option related to the placement in the sizer. Many widgets may have a default value of 3 in the "Border" property in the preferences dialog, see the section called “Preferences Dialog”. If you let widget to have a default border, also the wxALL option is checked.

The third tab, named "Widget", is different for each widget, and let you edit properties for the specific element you have selected.

Figure 3.8. Widget Properties

The set of option may be also quite complex, in case of widget that have a big lot of method and properties (grids, treeviews). In this case, wxGlade, greatly simplify the process of designing forms.

You can show or hide the properties window by the menu item View->Show Properties (Ctrl-P).

Editing properties

Editing properties is quite simple; Properties are represented by buttons, text boxes and checks. Usually they are refferred by the same name or symbol that you find writing C++ code.

Usually you get the changes in the design window in real time. In some cases you have to push he "Apply" button. For example, the wxNotebook widget show in his properties window a list of child wxPanels. You have to press the apply button to show changes you make when you add or remove panels.
