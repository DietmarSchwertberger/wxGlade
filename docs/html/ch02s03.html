Command line invocation

You can run wxGlade without parameters to start the GUI on a empty application as follow:

wxglade.py

Run wxGlade GUI on an existing application specifying the .wxg file as follow:

wxglade.py FILENAME.wxg

If you want only to generate the code without starting the GUI, use the -g or --generate-code option with the language as argument as follow:

wxglade.py -g LANG FILENAME.wxg

wxglade.py --generate-cod=LANG FILENAME.wxg

Possible values for LANG are "python", "XRC", "perl" or "C++".

You also can specify the destination of the generated code with -o or --output option:

wxglade.py -g LANG -o DESTINATION FILENAME.wxg

The DESTINATION argument can be a file or a directory. It is a file while the FILENAME.wxg generate a single file source code. It is a directory while the FILENAME.wxg generate multiple file source code.

This is the complete description of command line:

wxglade.py [OPTIONS] [FILENAME.wxg] OPTIONS: -g LANG, --generate-code=LANG Generate the code in the LANG language. Possible values for LANG are C++, python, perl, XRC. -o DEST, --output=DEST Only valid with the -g option. Specify the destination of the generation of code. The argument DEST can be a source code file or a directory.
